How it works
Step-by-Step Flow
This is exactly what happens every morning at 10 AM IST, in order.
1
GitHub wakes up the bot
GitHub Actions has a scheduler — like a cron job. Every day at 10 AM IST it automatically starts a fresh Ubuntu computer on GitHub's servers and runs our code.
Automated — no human needed
2
Opens Naukri in a hidden browser
We use Playwright — a tool that controls a real Chrome browser but invisibly (called "headless"). It goes to naukri.com/login and types in your email and password just like a human would.
Playwright + Stealth Plugin
3
Naukri sends an OTP to your Gmail
Because GitHub's server is a new/unknown device every time, Naukri asks for an OTP to verify it's really you. It sends a 6-digit code to your registered email with the subject "Your OTP for logging in Naukri account".
Security step by Naukri
4
Bot reads the OTP from Gmail automatically
This is the smartest part. Our script connects to your Gmail using IMAP (an email protocol) with an App Password. It searches your inbox for emails from the last 5 minutes with that exact subject, opens it, and extracts the 6-digit number.
IMAP + Gmail App Password
5
Types the OTP and clicks Verify
The bot types each digit of the OTP into Naukri's 6 input boxes and clicks the Verify button — exactly like a human would.
Auto login complete
6
Renames your resume with today's date
Before uploading, the script renames your PDF from resume.pdf to something like Saurav_Thakur_Updated_Resume_21Jun2026.pdf — so Naukri knows it's a fresh upload for today.
Date: auto-generated daily
7
Uploads the resume to your profile
Navigates to your Naukri profile page and uploads the renamed PDF. Your profile's "last updated" date is now today — putting you back at the top of recruiter searches.
✓ Done — profile refreshed

Credentials Setup
What You Need to Create
Three secrets stored safely in GitHub — never exposed in the code.
Secret Name What it is Example
NAUKRI_EMAIL Your Naukri login email address saurav@gmail.com
NAUKRI_PASSWORD Your Naukri account password MyPassword@123
GMAIL_APP_PASSWORD A special 16-character password generated by Google — not your real Gmail password abcdefghijklmnop
Where to add these: Go to your GitHub repo → Settings → Secrets and variables → Actions → New repository secret. Add each one by name.
One-time setup
How to Generate Gmail App Password
This takes about 2 minutes. You only do this once.
1
Turn on 2-Step Verification
Go to myaccount.google.com/security and make sure 2-Step Verification is switched ON. App Passwords only work when this is enabled.
2
Open App Passwords page
Go to myaccount.google.com/apppasswords — you may be asked to log in again.
3
Create a new App Password
In the text box, type Naukri Bot and click the Create button.
4
Copy the 16-character code
Google shows you a code that looks like: abcd efgh ijkl mnop — copy it immediately. Remove the spaces when saving it. This is your GMAIL_APP_PASSWORD.
5
Add it to GitHub Secrets
Paste the 16-char code (no spaces) as a GitHub secret named GMAIL_APP_PASSWORD. Done!
Why App Password and not your real Gmail password? Google blocks your real password when used by scripts for security. An App Password is a special code that only works for one specific app. You can delete it anytime without changing your main Gmail password.
The Smart Part
How OTP Reading Works
This is what makes the project unique — fully automatic OTP handling.
IMAP Protocol
IMAP is a standard email reading protocol. Think of it like a direct connection to your Gmail inbox — our script dials in, searches for the specific Naukri OTP email, reads it, and logs out. All within seconds.
5-Minute Safety Window
The script only looks at emails received in the last 5 minutes. This means it can never accidentally read an old OTP from a previous session or a different login attempt.
Exact Subject Match
It only picks emails with the exact subject: "Your OTP for logging in Naukri account" — so it never accidentally reads the wrong email, no matter how many emails are in your inbox.
Multi-Layer Extraction
The OTP is extracted using 5 different strategies — checking plain text, HTML content, raw MIME source — so it works even if Naukri changes their email format in the future.
